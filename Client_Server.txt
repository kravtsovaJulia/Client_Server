1) Прочитать про клиент-серверную архитектуру

Клиент-серверная архитектура - это обмен информацией между клиентом и сервером с помощью HTTP протокола.

2) Что такое HTTP и HTTPS

HTTP - это протокол с помощью которого устанавливаются правила обмена информацией между клиентом и сервером.

HTTPS - это расширенный протокол HTTP, более защищённый 

3) HTTP методы 

GET - получение данных, передается и находится в URL строке, у него нет тела ( body )
PUT -  изменения данных, оно имеет тело ( body ), изменение данных происходит по id 
POST -  сохраняет данные, оно имеет тело ( body )
DELETE - удаление данных, у него нет тела ( body ), удаление происходит по id 
PATCH - аналог PUT, но изменяет только часть данных
CONNECT - преобразует соединения запроса в прозрачный TCP/IP туннель
TRACE - возвращает полученный запрос так, что клиент может видеть, какую информацию промежуточные сервера добавляют или изменяют в запросе.
 HEAD - аналогичен методу GET, но в ответе сервера отсутствует тело.

4) HTTP статус коды сервера

1xx - информативный 
2xx - успех 
3xx - перенаправлен
4xx - ошибка на стороне клиента
5xx - ошибка на стороне сервера 

Популярные HTTP запросы:
200 OK - запрос выполнен успешно
201 Создано - был успешно создан ресурс, подтверждение успешного запроса PUT или POST
400 Bad Request - данные не прошли проверку или данные передаются в неправильном формате.
404 Not Found - требуемый ресурс не найден или не доступен.
401 Unauthorized - необходимо предоставить аутентификацию перед доступом к ресурсу.
409 Conflict - пользователь дважды создал один и тот же ресурс, из-за которого происходит конфликт.
500 Внутренняя ошибка сервера -  ошибка на стороне сервера 
502 Bad Gateway - сервер получил неверный ответ от другого сервера 


5) Что такое ядро браузера 

Ядро браузера -  это система, отвечающая за получение содержимого веб-страницы (HTML, XML, изображения и т. д.), организацию информации (например, добавление CSS) и вычисления режима отображения веб-страницы и вывод на монитор или принтер.

Mozilla FireFox - Gecko

7) Что такое API

API ( Application Programming Interface ) - это интерфейс с помощью которого, можно получить доступ к какому либо сервису  и запросить у него данные.

8. Что такое эндпоинты

Эндпоинт (конечная точка) - это адрес, на который отправляется сообщение. Некое соединение, которое соединяет серверные процессы с внешним интерфейсом.

9) URL (URI, URL, URN)

URL - адрес ресурса в сети, определяет место положение и способ обращения к нему.
URI - имя и адрес ресурса в сети, включает в себя URL и URN
URN - имя ресурса в сети, определяет только название ресурса, но не говорит, как к нему подключится.

10) Идемпотентные HTTP методы

 Идемпотентные HTTP методы -  это тогда, когда одни и те же запросы возвращают одинаковый результат.

Из спецификации HTTP идемпотентными считаются: 
GET
HEAD
TRACE

11) Безопасные HTTP методы

Безопасными считаются методы, которые, не меняют состояние сервера. 
GET
HEAD
TRACE

12) Иденфикация, Аутентификация, Авторизация

Иденфикация - процедура, выявляющая индентификатор, который однозначно определяет этот субьект в информационной системе.
Аутентификация - процедура проверки подлинности, например, сравнение введенного пароля и логина с имеющимся в базе данных.
Авторизация - предоставление определенному лицу или группе лиц права на выполнение определенных действий.


13) Что такое IP

IP " Интернет-протокол" - набор правил регулирующий формат данных, отправляемых через интернет или локальную сеть.
IP - адрес - это уникальный адрес, идентифицирующий устройство в интернете или локальной сети.

14) Что такое октеты в DNS

 DNS - это сервер на котором запущенно специальное программное обеспечение. 
Изначально этот сервер был создан для хранения таблицы DNS-записей вида: «имя домена» — «IP-адрес», но сейчас там содержатся и другие ресурсы, такие как: «MX», «TXT», «CNAME» и др.

Октет (информатика) — 8 бит (иначе: байт).
Октет в IP -  это каждая из 4 частей, записанных в десятичных числах с точками (например, 192.168.1.1).
Header — Заголовок DNS пакета, состоящий из 12 октет.

15) Что такое порт, сколько портов у Linux сервера

Порт - путеводитель по интернету.
HTTP - 80 порт.
HTTPS - 443 порт.
